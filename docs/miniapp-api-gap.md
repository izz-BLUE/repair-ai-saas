# 小程序接口适配分析与缺口

> 创建日期：2026-06-22
> 版本：V0.4.3
> 状态：📋 分析完成，标记缺口但不实现

## 一、现有接口适配表

### 1.1 客户侧公开接口

| 接口 | 当前用途 | 小程序复用 | 改造需求 | 缺口 | 优先级 |
|------|---------|-----------|---------|------|--------|
| `GET /api/public/{tenantCode}/portal-settings` | 获取企业门户配置（名称/主题色/Logo/启用状态） | ✅ 直接复用 | 无 | 无 | - |
| `POST /api/public/{tenantCode}/ai/chat` | AI 售后问答（问题+客户信息+产品类型） | ✅ 直接复用 | 无 | 无 | - |
| `POST /api/public/{tenantCode}/repair-requests` | 客户提交报修（姓名/电话/地址/产品/故障描述） | ✅ 可复用 | 后续增加图片字段 | 不支持报修图片上传 | P0 |
| `GET /api/public/{tenantCode}/tickets/query` | 按工单号+手机号查询进度 | ✅ 直接复用 | 无 | 无 | - |

### 1.2 师傅侧接口

| 接口 | 当前用途 | 小程序复用 | 改造需求 | 缺口 | 优先级 |
|------|---------|-----------|---------|------|--------|
| `POST /api/public/login` | 员工登录（返回 JWT + 角色 + 姓名） | ✅ 直接复用 | 无 | 无 | - |
| `GET /api/technician/tickets` | 我的工单列表（分页+状态筛选） | ✅ 直接复用 | 无 | 无 | - |
| `GET /api/technician/tickets/{id}` | 工单详情（含状态日志） | ✅ 直接复用 | 无 | 返回字段不含图片 | P0 |
| `PUT /api/technician/tickets/{id}/start` | 开始处理（ASSIGNED → IN_PROGRESS） | ✅ 直接复用 | 无 | 无 | - |
| `PUT /api/technician/tickets/{id}/complete` | 完成工单（维修结果+配件+费用+备注） | ✅ 可直接复用 | 后续增加图片字段 | 不支持完工图片上传 | P0 |

### 1.3 管理/平台侧接口（小程序不使用）

| 接口 | 当前用途 | 说明 |
|------|---------|------|
| `/api/platform/tenants` | 平台管理：租户 CRUD | 小程序不使用，仅 Web 平台管理使用 |
| `/api/admin/settings` | 企业设置：门户配置 | 小程序不使用，仅 Web 管理后台使用 |
| `/api/admin/tickets` | 后台工单管理 | 小程序不使用，仅 Web 管理后台使用 |

### 1.4 适配总结

| 类别 | 数量 | 状态 |
|------|------|------|
| 可**直接复用**的接口 | **6** | login, portal-settings, ai/chat, repair-requests, tickets/query, tickets list |
| **基本可用**、后续需增强的接口 | **3** | repair-requests（缺图片）, complete（缺图片）, ticket detail（缺图片） |
| **不需要**适配的接口 | **大量** | 所有 /api/admin/* , /api/platform/* |
| 小程序需要但**完全缺失**的接口 | **0** | MVP 阶段无 P0 缺失接口 |

> **结论：现有 API 对小程序 MVP 的适配度为 100%。** 所有 P0 功能都有对应接口，图片上传是后续增强，不影响 MVP 上线。

## 二、P0 缺口分析（MVP 必须）

### 2.1 客户报修图片上传

| 项目 | 说明 |
|------|------|
| **现状** | `POST /api/public/{tenantCode}/repair-requests` 无文件字段，仅接收文本 |
| **需求** | 客户报修时可拍照上传故障照片（1-3 张） |
| **方案** | 方案A：新增 `POST /api/public/{tenantCode}/repair-images` 上传接口 + repair-requests 中 `imageIds` 字段；方案B：multipart/form-data 混合上传（repair-requests 改为 multipart） |
| **推荐** | 方案A（先上传图片返回 ID，再提交报修时关联），与现有文档上传逻辑复用文件存储 |
| **风险** | 需要新增工单附件表 `ticket_attachment` 或复用现有文件存储机制 |

### 2.2 师傅完工图片上传

| 项目 | 说明 |
|------|------|
| **现状** | `PUT /api/technician/tickets/{id}/complete` 无文件字段 |
| **需求** | 师傅完成维修时上传维修前/后对比照片 |
| **方案** | 与报修图片上传共用同一文件上传接口，complete 请求中增加 `imageIds` 字段 |
| **推荐** | 统一定义 `ticket_attachment` 表，报修和完工都复用 |

### 2.3 工单详情返回图片

| 项目 | 说明 |
|------|------|
| **现状** | 工单查询/详情不返回图片信息（因无图片表） |
| **需求** | 客户和师傅都能看到报修/完工图片 |
| **方案** | 实现图片上传后，工单详情接口自动返回关联图片 URL 列表 |
| **依赖** | P0 缺口 2.1 和 2.2 |

## 三、P1 缺口分析（体验增强）

### 3.1 微信手机号授权绑定客户

| 项目 | 说明 |
|------|------|
| **现状** | 客户用明文手机号报修和查询 |
| **需求** | 小程序中通过微信授权获取手机号，减少手动输入 |
| **方案** | 新增 `POST /api/public/{tenantCode}/wx/phone-auth` 接口，接收微信加密手机号数据，解密后返回 |
| **备注** | 需要微信小程序认证账号 + 后端解密逻辑 |

### 3.2 微信 openid 绑定师傅账号

| 项目 | 说明 |
|------|------|
| **现状** | 师傅使用 tenantCode + username + password 登录 |
| **需求** | 师傅首次登录后绑定微信 openid，后续免密登录 |
| **方案** | `sys_user` 表新增 `wx_openid` 字段；新增 `POST /api/public/wx/technician-login` 接口（接收 wx.login() code，后端换取 openid 并查找绑定账号） |
| **备注** | 首次绑定仍需输入账号密码 |

### 3.3 订阅消息

| 项目 | 说明 |
|------|------|
| **现状** | 无消息推送机制 |
| **需求** | 师傅收到新派单通知、客户收到状态变更通知 |
| **方案** | 后端新增订阅消息发送逻辑；小程序端调用 `wx.requestSubscribeMessage` 获取授权 |
| **涉及接口** | 工单状态变更时触发消息推送（派单 → 师傅，状态变化 → 客户） |

### 3.4 企业二维码生成

| 项目 | 说明 |
|------|------|
| **现状** | 无二维码生成功能 |
| **需求** | 每个租户生成专属小程序码，客户扫码直达企业首页 |
| **方案** | 新增 `GET /api/admin/qrcode` 接口，调用微信 `wxacode.getUnlimited` API 生成小程序码，scene 参数携带 tenantCode |
| **备注** | 需要微信小程序 access_token 管理 |

### 3.5 请假/工时/工作统计

| 项目 | 说明 |
|------|------|
| **现状** | 无师傅工作统计 |
| **需求** | 师傅可查看自己的完工统计（本月完工数、评价等） |
| **方案** | 新增 `GET /api/technician/stats` 接口 |
| **备注** | 依赖工单评价体系 |

## 四、P2 缺口分析（长期规划）

| 缺口 | 说明 | 依赖 |
|------|------|------|
| 地图导航 | 师傅点击地址直接导航到客户位置（调用微信地图 API） | 前端调用 `wx.openLocation`，无需后端接口 |
| 语音输入 | AI 咨询支持语音转文字 | 微信小程序自带语音识别能力，前端实现 |
| OCR 识别 | 拍照识别设备型号、铭牌信息 | 需要第三方 OCR 服务或微信 OCR 插件 |
| 多端消息同步 | 管理员后台操作后小程序端实时更新 | 需要 WebSocket 或轮询 |
| 工单评价 | 客户对已完成工单打分评价 | 需要新增评价表和接口 |
| 配件库存管理 | 师傅可查询配件库存并申请配件 | 需要新增配件模块 |
