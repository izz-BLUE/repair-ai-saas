## 一、导入项目

### 1.1 打开微信开发者工具

下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

推荐 **稳定版 Stable Build**（Windows / macOS 均有）。

### 1.2 导入项目

1. 启动微信开发者工具，使用微信扫码登录
2. 点击"导入项目"（或 "+" → "导入项目"）
3. 填写以下信息：

| 字段 | 值 |
|------|-----|
| 项目名称 | `repair-miniapp` |
| 目录 | 选择 `miniapp-repair/` |
| AppID | 选择 **"测试号"** 或手动输入 `touristappid` |
| 开发模式 | 小程序（默认） |
| 后端服务 | 不使用云开发 |

> ⚠️ **不要使用真实小程序 AppID。** 验收阶段使用测试号或 `touristappid` 即可。真实 AppID 仅在申请正式小程序后替换 `project.config.json` 中的 `TODO_APPID`。

4. 点击"确定"，等待编译完成

### 1.3 配置不校验合法域名

由于开发阶段后端运行在 `localhost:8080`（非 HTTPS），需要关闭域名校验：

1. 开发者工具右上角「详情」→「本地设置」
2. 勾选 ✅ **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**

> 生产上线后必须移除该勾选，并配置真实 HTTPS 域名到小程序后台「开发 → 开发管理 → 开发设置 → 服务器域名」。

## 二、启动本地后端

小程序需要后端 API 运行在 `localhost:8080`：

```bash
# 终端 1：启动基础设施
cd repair-ai-saas
docker compose up -d

# 终端 2：启动 Java 后端
cd backend-java
mvn spring-boot:run
# 或使用本地 Maven：
# ..\.tools\apache-maven-3.9.9\bin\mvn.cmd spring-boot:run
```

验证后端可用：

```bash
curl http://localhost:8080/api/public/PLATFORM/portal-settings
# 返回 JSON 即表示后端正常
```

> baseUrl 配置在 `miniapp-repair/utils/config.js` 第 9 行：`baseUrl: 'http://localhost:8080'`。

## 四、师傅侧验收流程

所有师傅侧页面**需 TECHNICIAN 角色登录**。验收前需在后端准备测试师傅账号。

### 4.1 准备师傅账号

如果还没有师傅账号，用平台管理员创建：

1. 使用平台管理员登录 Web 管理后台
2. 进入员工管理 → 新建员工
3. 角色选择 **TECHNICIAN**
4. 记录：企业编码、用户名、密码

或在 Java 后端 Seed 数据中已有师傅账号（见 `init-demo-data.sh`）。

### 4.2 师傅登录（pages/technician/login）

**路径：** `pages/technician/login/login`
**也可从首页"我是维修师傅"进入。**

| 验收项 | 预期 |
|--------|------|
| 登录表单 | 企业编码 + 用户名 + 密码三个输入框 |
| 空值校验 | 任一为空 → toast 提示 |
| 错误登录 | 错误密码 → toast 显示错误信息（由后端返回） |
| 非师傅角色 | 如果账号不是 TECHNICIAN → toast 提示"仅维修师傅可登录师傅端" |
| 成功登录 | TECHNICIAN 账号登录成功 → 跳转到 `pages/technician/tickets/tickets` |
| token 持久化 | 关闭小程序再打开，如果 token 未过期 → 直接进入工作台 |
| 401 处理 | token 过期后请求 → 自动清 token 跳转登录页 |
| 已登录跳过 | 已登录的师傅再次进入 login 页 → 自动跳转工单列表 |

### 4.3 我的工单（pages/technician/tickets）

**登录后自动进入。**

| 验收项 | 预期 |
|--------|------|
| 状态筛选 | 横向滚动 Tab：全部/已派单/处理中/已完成/已关闭 |
| Tab 切换 | 点击 Tab → 列表刷新，选中态显示主题色填充 |
| 工单卡片 | 每张卡片显示：状态标签、工单号、客户名、产品类型、地址、时间 |
| 卡片点击 | 点击卡片进入工单详情 |
| 空状态 | 无工单时显示"暂无工单"图标+文字 |
| 仅看到自己 | 确认列表中只有分配给当前师傅的工单 |

### 4.4 工单详情（pages/technician/ticket-detail）

**从工单列表点击卡片进入。**

| 验收项 | 预期 |
|--------|------|
| 状态横幅 | 顶部彩色横幅显示当前状态（如"已派单"蓝色） |
| 客户信息卡 | 姓名、电话（可拨打）、地址（可复制） |
| 工单信息卡 | 工单号（monospace）、产品类型、故障描述、优先级、创建时间 |
| 时间线 | 竖线+圆点时间线，显示历史状态流转 |

## 五、常见问题

### Q1：页面白屏，console 显示 `request:fail url not in domain list`

**原因：** 未关闭"不校验合法域名"设置。

**解决：** 开发者工具右上角「详情」→「本地设置」→ 勾选 ✅ **不校验合法域名...**

### Q2：请求返回 500 / 网络错误

**原因：** 后端未启动或 baseUrl 配置错误。

**解决：**
1. 确认 Java 后端在 `localhost:8080` 运行
2. 检查 `miniapp-repair/utils/config.js` 中 `baseUrl` 是否正确
3. 在浏览器访问 `http://localhost:8080/api/public/PLATFORM/portal-settings` 验证

### Q3：登录成功但页面不跳转

**原因：** token 存储失败或角色校验拦截。

**解决：**
1. console 打日志检查 `auth.saveAuth` 调用是否成功
2. 确认账号角色为 `TECHNICIAN`
3. 检查 `auth.checkTechnicianAccess` 返回值

### Q4：客户侧首页显示"缺少企业编码"

**原因：** tenantCode 未传入或未存储。

**解决：**
1. 确保 URL 中有 `?tenantCode=xxx` 参数
2. 或通过编译模式设置启动参数（详情 → 编译模式 → 添加编译模式 → 启动页面 + 启动参数）

### Q5：`project.config.json` 中 `appid` 字段报错

**原因：** 使用了 `TODO_APPID` 占位但正式编译需要合法 AppID。

**解决：** 使用"测试号"导入项目，测试号会自动分配一个临时 AppID。

## 六、生产上线前必须替换

| 项目 | 位置 | 当前值 | 替换为 |
|------|------|--------|--------|
| 小程序 AppID | `project.config.json` → `appid` | `TODO_APPID` | 正式小程序 AppID |
| 后端域名 | `utils/config.js` → `baseUrl` | `http://localhost:8080` | `https://api.你的域名.com` |
| 不校验合法域名 | 开发者工具设置 | ✅ 勾选 | ❌ 取消勾选 |
| request 合法域名 | 微信公众平台 → 服务器域名 | 无 | `https://api.你的域名.com` |
| JWT_SECRET | `.env` | 默认值 | `openssl rand -base64 32` 生成 |
| API Key | `.env` | 默认值 | 真实 DeepSeek / OpenAI API Key |

> ⚠️ **永远不提交**真实 AppID、密钥、域名到代码仓库。生产配置使用 `.env` 环境变量注入，小程序配置在上线前手动替换。
