## 2. 租户生命周期设计

### 2.1 状态定义（纯字符串 → TenantStatus 枚举）

```
┌──────────┐   创建租户    ┌──────────┐   到期自动/手动   ┌──────────┐
│  TRIAL   │ ────────────→ │  ACTIVE  │ ────────────────→ │ EXPIRED  │
│ (试用期)  │               │ (正式使用) │                    │ (已到期)  │
└────┬──────┘               └────┬──────┘                    └────┬──────┘
     │                           │                                │
     │ 平台手动                   │ 平台手动                        │ 平台手动
     │                           │                                │
     ▼                           ▼                                ▼
┌──────────┐               ┌──────────┐                    ┌──────────┐
│SUSPENDED │               │SUSPENDED │                    │  CLOSED  │
│ (暂停中)  │               │ (暂停中)  │                    │ (已关闭)  │
└──────────┘               └──────────┘                    └──────────┘
     │                           │
     │ 平台恢复                   │ 平台恢复
     ▼                           ▼
  原状态                       原状态
```

### 2.2 状态说明

| 状态 | 值 | 含义 | 触发方式 |
|------|------|------|------|
| `TRIAL` | `"TRIAL"` | 试用中 | 创建租户时默认（取代当前 `"ACTIVE"`） |
| `ACTIVE` | `"ACTIVE"` | 正式付费/长期使用 | 平台管理员手动切换 |
| `EXPIRED` | `"EXPIRED"` | 已到期 | 过期后首次请求时自动转换（或定时任务批量转换） |
| `SUSPENDED` | `"SUSPENDED"` | 被平台暂停 | 平台管理员手动暂停 |
| `CLOSED` | `"CLOSED"` | 永久关闭/注销 | 平台管理员手动关闭 |

### 2.3 状态转换规则

```java
// 合法转换（TenantStatus 枚举控制）
TRIAL     → ACTIVE       // 平台管理员确认转为正式
TRIAL     → SUSPENDED    // 违规暂停
TRIAL     → EXPIRED      // 试用到期
TRIAL     → CLOSED       // 平台关闭

ACTIVE    → EXPIRED      // 到期
ACTIVE    → SUSPENDED    // 违规暂停
ACTIVE    → CLOSED       // 平台关闭

EXPIRED   → ACTIVE       // 续期恢复
EXPIRED   → SUSPENDED    // 过期后违规暂停
EXPIRED   → CLOSED       // 过期后关闭

SUSPENDED → TRIAL        // 恢复试用
SUSPENDED → ACTIVE       // 恢复正式
SUSPENDED → CLOSED       // 暂停后关闭
```

### 2.4 兼容性处理

当前代码中散落着多处 `"ACTIVE".equals(tenant.getStatus())` 硬编码（至少有 4 处）。V0.7.0 需要：

1. 新建 `TenantStatus` 枚举，包含 `isUsable()` 方法
2. 在 `Tenant` 实体中添加 `isUsable()` 便捷方法（基于 status 非 EXPIRED/SUSPENDED/CLOSED）
3. 重构所有 `"ACTIVE".equals(...)` 为 `tenant.isUsable()` 调用（含 TRIAL 和 ACTIVE）
4. 数据库现有 `"ACTIVE"` 记录保持不变（枚举的 `ACTIVE.name()` 也是 `"ACTIVE"`）
5. 现有 `"INACTIVE"` 记录需要手动迁移为 `"SUSPENDED"`（Flyway V8）

**⚠️ 风险点：** 数据库现有 status = `"INACTIVE"` 的记录——当前语义模糊（可能是禁用也可能是旧数据），迁移时需确认。

## 3. 套餐模型设计

### 3.1 套餐字段（直接加在 tenant 表上）

**不在 v0.7.0 用独立 plan 表。** 理由：
- MVP 阶段套餐数量极少（≤3 种），独立表增加 JOIN 和迁移复杂度
- 直接用 tenant 列存储套餐标识，未来扩展时迁移到独立表成本可控
- 保持与现有 `maxKnowledgeBases`/`maxDocuments`/`maxAiDailyCalls` 的列级存储模式一致

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `plan_code` | VARCHAR(50) NULL | 套餐标识码 | `"TRIAL"`（新租户） |
| `plan_name` | VARCHAR(100) NULL | 套餐名称（展示用） | `"试用版"` |
| `max_users` | INT NULL | 员工账号上限（NULL=不限） | `5`（试用） |
| `max_technicians` | INT NULL | 师傅账号上限（NULL=不限） | `3`（试用） |
| `max_knowledge_bases` | INT NULL | 知识库上限 | ✅ 已有 |
| `max_documents` | INT NULL | 文档上限 | ✅ 已有 |
| `max_ai_daily_calls` | INT NULL | AI 日调用上限 | ✅ 已有 |
| `ticket_monthly_limit` | INT NULL | 月工单上限（NULL=不限） | `50`（试用） |
| `trial_end_at` | DATETIME NULL | 试用到期日（NULL=非试用） | 创建日+14天 |
| `expired_at` | DATETIME NULL | 服务到期日 | ✅ 已有 |

### 3.2 内置套餐定义（代码常量或配置文件）

### 3.3 套餐切换行为

- 平台管理员在 `PUT /api/platform/tenants/{id}` 中修改 `planCode`
- 修改 `planCode` **不会自动刷新额度字段**（额度字段可以独立调整）
- 前端可选提供「应用套餐预设」按钮，一键填充所有额度字段
- `trialEndAt` 在租户创建时自动设为 `now() + 14 days`，切换为正式套餐时自动清除

## 5. 租户到期后行为规则

### 5.1 按角色分场景

| 行为 | TRIAL | ACTIVE | EXPIRED | SUSPENDED |
|------|:---:|:---:|:---:|:---:|
| 后台登录（ADMIN/DISPATCHER） | ✅ | ✅ | ✅（仅查看） | ❌ |
| 师傅端登录（TECHNICIAN） | ✅ | ✅ | ❌ | ❌ |
| 客户报修（公开门户） | ✅ | ✅ | ❌ | ❌ |
| 客户查询进度（公开门户） | ✅ | ✅ | ✅ | ❌ |
| 后台创建/编辑工单 | ✅ | ✅ | ❌ | ❌ |
| 后台派单/处理工单 | ✅ | ✅ | ❌ | ❌ |
| AI 咨询（公开门户） | ✅ | ✅ | ✅ | ❌ |
| 后台 AI 对话 | ✅ | ✅ | ✅ | ❌ |
| 知识库管理 | ✅ | ✅ | ✅（只读） | ❌ |
| 员工管理 | ✅ | ✅ | ❌ | ❌ |

### 5.2 规则说明

**EXPIRED（已到期）设计原则：**
- 保留只读访问：后台可登录查看历史数据，AI 可继续咨询（不中断客户自助服务体验）
- 禁止写入：不接收新报修、不创建/编辑工单、不管理员工
- 客户可查询已有工单进度
- 宽限期：到期后 7 天内不改变 status（仍是 EXPIRED），7 天后可自动转为 CLOSED（可选，v0.7.0 不做自动转换，仅记录设计意图）
- `isTenantAvailable()` 保持现有逻辑：EXPIRED 时返回 false（阻止业务操作）
- 但 `getActiveUserForAuth()` 需调整：EXPIRED 时允许登录但不允许写操作

**SUSPENDED（被暂停）设计原则：**
- 全面禁止：不可登录、不可报修、不可查询
- 返回统一的"企业服务暂不可用"错误
- 区别于 EXPIRED 的"该企业服务已到期"错误

### 5.3 实现方式

在 `TenantStatus` 枚举中定义能力方法：

```java
public enum TenantStatus {
    TRIAL, ACTIVE, EXPIRED, SUSPENDED, CLOSED;

    /** 是否允许任何后端用户登录 */
    public boolean allowsLogin() {
        return this == TRIAL || this == ACTIVE || this == EXPIRED;
    }

    /** 是否允许写操作（创建/编辑工单、管理员工等） */
    public boolean allowsWrite() {
        return this == TRIAL || this == ACTIVE;
    }

    /** 是否允许客户报修 */
    public boolean allowsPublicRepair() {
        return this == TRIAL || this == ACTIVE;
    }

    /** 是否允许客户查询工单 */
    public boolean allowsPublicQuery() {
        return this != SUSPENDED && this != CLOSED;
    }

    /** 兼容旧 isTenantAvailable 语义 */
    public boolean isUsable() {
        return this == TRIAL || this == ACTIVE;
    }
}
```

### 5.4 试用到期提醒

- `trialEndAt` 前 3 天，后台首页展示提醒横幅："您的试用期将于 X 天后到期"
- 到期当天展示："试用期今日到期，请联系平台升级"
- 提醒仅展示，不阻断操作（TRIAL 状态在 trialEndAt 过后才自动转 EXPIRED）
- 自动转换时机：首次请求时检测 `trialEndAt` 已过 → 自动将 status 从 TRIAL 改为 EXPIRED

## 6. 额度拦截点设计

### 6.1 拦截点总览

| # | 操作 | API 路径 | 限制字段 | 检查方式 | 现有状态 |
|---|------|------|------|------|:---:|
| 1 | 创建员工 | `POST /api/admin/users` | `maxUsers` | count ACTIVE users ≤ maxUsers | ❌ 新增 |
| 2 | 创建师傅 | `POST /api/admin/users`（role=TECHNICIAN） | `maxTechnicians` | count ACTIVE technicians ≤ maxTechnicians | ❌ 新增 |
| 3 | 创建知识库 | `POST /api/admin/knowledge-bases` | `maxKnowledgeBases` | count knowledge_bases ≤ maxKnowledgeBases | ✅ 已有 |
| 4 | 上传文档 | `POST /api/admin/knowledge-bases/{id}/documents` | `maxDocuments` | count documents ≤ maxDocuments | ❌ 新增 |
| 5 | AI Chat（公开+后台） | `POST /api/public/{code}/ai/chat`、`POST /api/admin/ai/chat` | `maxAiDailyCalls` | today's count ≤ maxAiDailyCalls | ✅ 已有 |
| 6 | 客户报修 | `POST /api/public/{code}/repair-requests` | `ticketMonthlyLimit` | this month's count ≤ ticketMonthlyLimit | ❌ 新增 |

### 6.2 拦截实现模式

```java
// 通用拦截器：检查租户额度
public class QuotaChecker {
    
    /** 检查并抛出 BusinessException，附带友好错误信息 */
    public static void checkQuota(Long currentCount, Integer maxAllowed, String resourceName) {
        if (maxAllowed != null && currentCount >= maxAllowed) {
            throw new BusinessException(ResultCode.FORBIDDEN,
                    String.format("%s数量已达上限（%d），如需扩容请联系平台", resourceName, maxAllowed));
        }
    }
}
```

### 6.3 具体实现位置

| 操作 | Service 方法 | 新增代码 |
|------|------|------|
| 创建员工 | `SysUserService.createUser()` | 开头：count ACTIVE users for tenantId → checkQuota(maxUsers) |
| 上传文档 | `KnowledgeDocumentService.upload()` | 开头：count documents for tenantId → checkQuota(maxDocuments) |
| 客户报修 | `TicketService.createPublicTicket()` | 开头：count this-month tickets for tenantId → checkQuota(ticketMonthlyLimit) |

### 6.4 工单月度计数策略

- 统计口径：`createdAt >= 当月1日 00:00:00` 的工单数
- 使用 `repair_ticket` 表 `selectCount`（MyBatis-Plus @TableLogic 自动排除 deleted）
- 不新建 `ticket_usage_monthly` 表（MVP 不需要月统计历史表）

### 6.5 用户数计数策略

- 统计口径：`count(*) from sys_user where tenant_id = ? and status = 'ACTIVE' and deleted = 0`
- 停用用户不计入额度
- 创建时检查，停用/启用时不检查

### 6.6 师傅数计数策略

- 统计口径：`count(*) from sys_user where tenant_id = ? and role = 'TECHNICIAN' and status = 'ACTIVE' and deleted = 0`
- 和 maxUsers **独立计数**：一个用户同时占用 maxUsers 和 maxTechnicians（如果角色是师傅）

## 附录 A：TenantStatus 枚举完整定义（伪代码）

```java
package com.repair.ai.saas.module.tenant.enums;

import com.repair.ai.saas.common.BusinessException;
import com.repair.ai.saas.common.ResultCode;
import java.util.Set;

public enum TenantStatus {
    TRIAL,
    ACTIVE,
    EXPIRED,
    SUSPENDED,
    CLOSED;

    /** 允许后端用户登录（ADMIN/DISPATCHER/TECHNICIAN 等） */
    public boolean allowsLogin() {
        return this == TRIAL || this == ACTIVE || this == EXPIRED;
    }

    /** 允许写操作（创建/编辑工单、管理员工等） */
    public boolean allowsWrite() {
        return this == TRIAL || this == ACTIVE;
    }

    /** 允许公开报修 */
    public boolean allowsPublicRepair() {
        return this == TRIAL || this == ACTIVE;
    }

    /** 允许公开查询工单 */
    public boolean allowsPublicQuery() {
        return this != SUSPENDED && this != CLOSED;
    }

    /** 兼容旧 isTenantAvailable 语义（业务操作可用） */
    public boolean isUsable() {
        return this == TRIAL || this == ACTIVE;
    }

    /** 合法状态转换 */
    public boolean canTransitionTo(TenantStatus target) {
        return switch (this) {
            case TRIAL    -> target == ACTIVE || target == SUSPENDED || target == EXPIRED || target == CLOSED;
            case ACTIVE   -> target == EXPIRED || target == SUSPENDED || target == CLOSED;
            case EXPIRED  -> target == ACTIVE || target == SUSPENDED || target == CLOSED;
            case SUSPENDED -> target == TRIAL || target == ACTIVE || target == CLOSED;
            case CLOSED   -> false; // 终态
        };
    }

    /** 验证并执行状态转换 */
    public static void validateTransition(TenantStatus from, TenantStatus to, String tenantCode) {
        if (from == CLOSED) {
            throw new BusinessException(ResultCode.VALIDATION_ERROR, "已关闭的租户不能修改状态");
        }
        if (!from.canTransitionTo(to)) {
            throw new BusinessException(ResultCode.VALIDATION_ERROR,
                String.format("不允许从 %s 转换为 %s", from.name(), to.name()));
        }
    }
}
```

## 附录 C：风险与缓解措施

| 风险 | 等级 | 缓解措施 |
|------|:---:|------|
| 现有 INACTIVE 租户含义不明，迁移为 SUSPENDED 可能误伤 | 中 | Flyway 前手动检查 INACTIVE 租户列表，按实际情况决定 |

| 额度检查影响现有租户正常使用 | 高 | V8 迁移为所有 ACTIVE 租户设默认试用额度，确保不低于当前实际用量 |
| EXPIRED 下 AI 仍可用但不计费，可能被滥用 | 低 | AI 日调用上限在 EXPIRED 下仍然生效 |
| ticketMonthlyLimit 实时 COUNT 性能 | 低 | repair_ticket 表有 tenant_id 索引，月量级不超过万级 |
