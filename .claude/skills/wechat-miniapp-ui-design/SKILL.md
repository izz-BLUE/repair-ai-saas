# 售后维修小程序 UI 设计规范

> 适用于 repair-ai-saas 独立售后维修小程序（V0.5.0+）。
> 技术栈：原生微信小程序 WXML / WXSS / JavaScript。

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **移动端唯一** | 所有页面按 375px 基准设计，不使用 PC 布局 |
| **原生微信** | 使用微信原生组件和交互习惯，不搬运 Ant Design Web 视觉 |
| **客户 vs 师傅** | 两种角色使用不同的视觉策略（见第二节） |
| **主题色驱动** | 客户侧主色调从 `GET /api/public/{tenantCode}/portal-settings` 的 `themeColor` 读取 |
| **状态可见** | 每种工单状态有明确的颜色和标签，一眼可辨 |
| **安全第一** | 不提交真实 AppID、密钥、域名；使用占位符 `TODO_APPID` |

---

## 二、角色视觉策略

### 2.1 客户侧（轻量、可信、服务感）

- **配色**：白底 + 主题色点缀（按钮、强调文字）
- **卡片**：大圆角（16rpx）、浅阴影、充足留白
- **按钮**：主题色实底，高度 88rpx，圆角 12rpx，字号 32rpx
- **状态标签**：彩色圆点 + 文字，放在卡片右上角
- **入口卡片**：图标 + 标题 + 描述，3 张卡片垂直排列
- **聊天**：气泡式，左侧 AI 灰底，右侧用户主题色底

### 2.2 师傅侧（高效、醒目、少步骤、大触控）

- **配色**：白底 + 系统蓝 #1677ff 为主操作色
- **卡片**：小圆角（8rpx）、1px 边框、紧凑间距
- **按钮**：大触控目标，最小高度 88rpx，间距足够防误触
- **状态标签**：大号彩色 Tag，放在卡片顶部
- **筛选项**：横向滚动 Tab 或 Segmented 控件
- **底部操作栏**：fixed bottom，1-2 个大按钮，带 safe-area 适配

## 三、通用规范

### 3.1 状态体系

| 状态 | 颜色 | 用途 |
|------|------|------|
| 待处理 PENDING | `#faad14` (橙) | 等待派单 |
| 已派单 ASSIGNED | `#1677ff` (蓝) | 等待师傅处理 |
| 处理中 IN_PROGRESS | `#722ed1` (紫) | 师傅正在维修 |
| 已完成 COMPLETED | `#52c41a` (绿) | 维修完成 |
| 已回访 FOLLOWED_UP | `#13c2c2` (青) | 已回访 |
| 已关闭 CLOSED | `#8c8c8c` (灰) | 工单关闭 |
| 已取消 CANCELLED | `#ff4d4f` (红) | 已取消 |

**WXSS 变量定义（app.wxss）：**
```css
page {
  --color-pending: #faad14;
  --color-assigned: #1677ff;
  --color-in-progress: #722ed1;
  --color-completed: #52c41a;
  --color-followed-up: #13c2c2;
  --color-closed: #8c8c8c;
  --color-cancelled: #ff4d4f;
  --color-primary: #1677ff;
  --color-text: #1a1a1a;
  --color-text-secondary: #666666;
  --color-text-hint: #999999;
  --color-bg: #f5f5f5;
  --color-card: #ffffff;
  --color-border: #e8e8e8;
  --radius-sm: 8rpx;
  --radius-md: 12rpx;
  --radius-lg: 16rpx;
  --btn-height: 88rpx;
  --font-title: 32rpx;
  --font-body: 28rpx;
  --font-caption: 24rpx;
}
```

### 3.2 安全区适配

所有底部固定操作栏必须适配 iPhone X+ 安全区：

```css
.fixed-bottom {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  padding-bottom: calc(24rpx + env(safe-area-inset-bottom));
  background: #fff;
  border-top: 1rpx solid var(--color-border);
}
```

### 3.3 通用状态页面

| 状态 | 表现 |
|------|------|
| **loading** | 居中 `wx:loading` 或骨架屏 |
| **empty** | 居中图标 + "暂无数据" 灰色文字 |
| **error** | 居中图标 + 错误信息 + "重试" 按钮 |
| **disabled** | 按钮灰色不可点击，opacity 0.4 |
| **success** | `wx.showToast({ icon: 'success' })` + 自动返回 |

### 3.4 按钮尺寸

| 场景 | 高度 | 字号 | 圆角 |
|------|------|------|------|
| 主操作按钮 | 88rpx | 32rpx | 12rpx |
| 次要按钮 | 72rpx | 28rpx | 8rpx |
| 行内小按钮 | 56rpx | 24rpx | 6rpx |

### 3.5 表单规范

- 标签：32rpx，颜色 `--color-text`
- 输入框：高度 88rpx，背景 `#f7f8fa`，圆角 12rpx
- 必填标记：红色 `*` 前缀
- 错误提示：红色文字 24rpx，紧贴输入框下方
- 提交按钮：底部 fixed，安全区适配

## 五、禁止事项

| ❌ 禁止 | ✅ 替代 |
|--------|--------|
| 使用 Ant Design Web 组件视觉 | 微信原生组件 + 自定义 WXSS |
| PC 后台数据表格风格 | 移动端卡片式布局 |
| 照搬 H5 的 antd Tag/Badge | 自定义 `<view class="tag tag-pending">` |
| 提交真实 AppID | `project.config.json` 使用 `touristappid` 或 `TODO_APPID` |
| 提交真实域名 | `utils/config.js` 使用 `http://localhost:8080` 占位 |
| 提交真实密钥/secret | 永远不写入代码，用注释标注 `<!-- TODO: 配置 -->` |
| 使用 Taro/uni-app 框架 | 原生 WXML/WXSS/JS |
| 忽略 safe-area | 所有 bottom fixed 元素加 `env(safe-area-inset-bottom)` |
| 照搬 PC 端的深色侧边栏 | 小程序无侧边栏，使用顶部导航或页面内跳转 |

## 六、验收清单

开发完成后逐项确认：

- [ ] 所有页面在 375px 宽度下正常显示
- [ ] 客户侧使用 portal-settings 返回的 themeColor
- [ ] 工单状态颜色与规范一致
- [ ] 底部固定栏适配 iPhone X+ 安全区
- [ ] 按钮高度 ≥ 88rpx（主操作）
- [ ] 有 loading / empty / error 状态处理
- [ ] 无真实 AppID / 密钥 / 域名
- [ ] 无 Ant Design 组件引用
- [ ] 无 PC 布局（max-width 不设 > 480px）
- [ ] 聊天页气泡样式正确
- [ ] 师傅登录页 401 自动清 token 跳转
